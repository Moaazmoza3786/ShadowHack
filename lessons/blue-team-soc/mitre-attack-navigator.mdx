# MITRE ATT&CK Navigator

<InfoBox type="tip" title="Industry Standard">
MITRE ATT&CK is the **universal language** for describing adversary behavior. Every security vendor, every threat report, and every mature SOC uses it.
</InfoBox>

## What is MITRE ATT&CK?

**ATT&CK** stands for **Adversarial Tactics, Techniques, and Common Knowledge**.

It's a knowledge base of adversary behavior based on real-world observations:
- **14 Tactics** (adversary goals)
- **200+ Techniques** (methods to achieve goals)
- **500+ Sub-techniques** (specific implementations)
- **100+ Groups** (threat actors)
- **500+ Software** (malware and tools)

### Tactics Overview

| # | Tactic | Goal | Example Techniques |
|---|--------|------|-------------------|
| 1 | **Reconnaissance** | Gather info | Active Scanning, Search Victim |
| 2 | **Resource Development** | Build infrastructure | Acquire Infrastructure, Develop Capabilities |
| 3 | **Initial Access** | Get into the network | Phishing, Valid Accounts, Exploit Public Apps |
| 4 | **Execution** | Run malicious code | PowerShell, Command Line, WMI |
| 5 | **Persistence** | Maintain access | Registry Run Keys, Scheduled Tasks |
| 6 | **Privilege Escalation** | Get higher permissions | UAC Bypass, Exploitation |
| 7 | **Defense Evasion** | Avoid detection | Obfuscation, Disable Security Tools |
| 8 | **Credential Access** | Steal credentials | Mimikatz, Keylogging, Brute Force |
| 9 | **Discovery** | Learn the environment | Network Scan, Account Discovery |
| 10 | **Lateral Movement** | Move to other systems | RDP, SMB, WinRM |
| 11 | **Collection** | Gather target data | Data from Local System, Screen Capture |
| 12 | **Command & Control** | Communicate with implants | HTTP/S, DNS, Web Service |
| 13 | **Exfiltration** | Steal data | Exfil Over C2, Cloud Storage |
| 14 | **Impact** | Destroy or manipulate | Ransomware, Data Destruction |

## Using the ATT&CK Navigator

The **ATT&CK Navigator** is a web-based tool for visualizing and working with the matrix.

<InfoBox type="info" title="Access Navigator">
ğŸ”— https://mitre-attack.github.io/attack-navigator/
</InfoBox>

### Key Features

1. **Create Layers** - Custom views of the matrix
2. **Color Techniques** - Highlight based on your criteria
3. **Score Techniques** - Rate coverage or priority
4. **Add Metadata** - Notes, links, comments
5. **Compare Layers** - Overlay multiple views

## Best Practices

1. **Start with threat actors** relevant to your industry
2. **Focus on high-frequency techniques** (PowerShell, Command Line, etc.)
3. **Update regularly** as you add new detections
4. **Share with your team** - Navigator layers are portable
5. **Use for tabletop exercises** - Walk through attack chains

<InfoBox type="warning" title="Don't Aim for 100%">
No organization can detect everything. Prioritize based on **threat relevance** and **business risk**.
</InfoBox>
