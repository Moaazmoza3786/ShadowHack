
# Enumerating Web Servers: Nikto & Gobuster

## The Web as the Primary Attack Surface
Web applications are the most common entry point for attackers because they are *designed* to be accessible to the public.

In this phase, we move from network-level scanning to application-level enumeration.

## 1. Nikto: The Web Vulnerability Scanner
Nikto is a classic Perl script that scans web servers for:
-   Outdated server software.
-   Dangerous files (e.g., `test.php`, `backup.sql`).
-   Misconfigurations (e.g., Missing HTTP Headers).

## 2. Directory Busting (Gobuster / Dirbuster)
Web servers often host hidden files that aren't linked from the homepage. 
**Directory Busting** is the process of brute-forcing these URL paths using a wordlist.

### Tool: Gobuster
A fast, Go-based directory brute-forcer.

**Command Structure:**
`gobuster dir -u <URL> -w <WORDLIST> -x <EXTENSIONS>`

### Analyzing the Output
*   `/admin` (301): Redirects to the admin panel. A prime target for brute-force attacks.
*   `/config.php`: Likely contains database credentials.
*   `/robots.txt`: Often lists "disallowed" directories that the admin doesn't want Google to index (which makes them interesting to us).

## 3. Robots.txt Analysis
Always check `http://target.com/robots.txt`. It's a goldmine of forbidden paths.

<CodeBlock language="text" showLineNumbers code={`User-agent: *
Disallow: /admin/
Disallow: /backup/
Disallow: /private/
Disallow: /dev/
`} />

In the example above, the admin has kindly provided us with a list of sensitive folders to attack.
